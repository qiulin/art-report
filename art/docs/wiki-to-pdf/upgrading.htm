Upgrading ART

Take the following steps to upgrade an existing ART installation to a more current version.

  • Download the latest ART version
  • Extract the zip package to a temporary location
  • Review the changelog.txt file to have an idea of what has changed
  • Backup the ART directory contained on the application server
  • Backup the ART repository (database)
  • Connect to the ART repository and run any database upgrade scripts included in the download package to bring the ART database up to date
  • Deploy the new art.war file to the application server. There are a number of ways to do this. You can do the following if using Apache Tomcat.
    • Stop Tomcat
    • Delete the art directory and art.war file in the TOMCAT_HOME\webapps directory
    • Delete ART's Tomcat cache by deleting the art directory in the TOMCAT_HOME\work\Catalina\localhost directory
    • Copy the new art.war file to the webapps directory
    • Start Tomcat
  • Copy the file art\WEB-INF\art.properties from the backup ART directory to the new version's WEB-INF directory.
  • Copy the contents of the backup art\WEB-INF\templates directory to the new version's templates directory

  • If you are using JasperReports templates and the new ART version has included an updated jasperreports engine, delete all the .jasper files contained in the templates directory so that they will be recompiled with the new engine. Alternatively, if you have Apache Ant set up, run the following commands

    cd art
ant clean-jasperreports
ant compile-jasperreports

  • Copy the contents of the backup art\export directory to the new version's export directory

  • Restart Tomcat
  • Log in to ART as an administrator. From the Admin Console, go to the Settings page and click on the Submit button to update application settings.
  • Clear your browser's cache e.g. for Firefox, use the Clear Recent History menu.

The upgrade is complete. All users, queries, scheduled jobs etc should continue to work as before.
